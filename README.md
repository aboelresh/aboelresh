Good software should become easier to maintain as it grows, not harder. That is the standard I hold my work to. I build backend systems with PHP and Laravel, focusing on clean architecture, honest API design, and code that the next engineer — or future me — can read without frustration.

Alongside engineering, I teach programming. Explaining something precisely requires understanding it precisely. Teaching keeps my fundamentals sharp and my assumptions honest.

I am actively expanding into Linux, networking, Docker, and cloud infrastructure — because understanding the environment my software runs in makes me a better engineer, not just a more marketable one.

## Philosophy

Understanding before memorization. Complexity is not a sign of quality.

Clean architecture outlasts clever code. A system that is easy to change is more valuable than one that is impressive to read once.

Teaching is not separate from engineering. Explaining something precisely requires understanding it precisely.

Technology exists to solve real problems. If a tool does not serve that, it is noise.

---

## Engineering Principles

- Design for maintainability from the start.
- Prefer clarity over cleverness.
- Document decisions, not just implementations.
- Build first, optimize when there is evidence.
- Understand the problem before adopting a tool.
